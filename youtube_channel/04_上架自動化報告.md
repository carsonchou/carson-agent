## ② 修了哪些問題（共 6 項）

### 【嚴重・會直接讓流水線中斷】串接介面對不上

**問題 1：`run_all.py` ③ 剪輯階段呼叫 `make_video.py` 用錯介面**
- 原本：`make_video.py <output/{slug}.mp3>`（當成位置參數）。
- 實際：`make_video.py` **沒有位置參數**，只吃 `--slug`／`--audio`／`--script`。
- 後果：argparse 直接報「unrecognized arguments」，③ 一定失敗。
- 修正：改成 `make_video.py --slug {slug} --output-dir <dir> [--config ...] [--dry-run]`，
  由 make_video 自行推導 `output/{slug}.mp3/.md/.voice.txt → .mp4`，與編排器完全一致。

**問題 2：`run_all.py` ④ 上傳階段呼叫 `upload_youtube.py` 用錯介面**
- 原本：`upload_youtube.py <output/{slug}.mp4>`（把完整 mp4 路徑當位置參數）。
- 實際：`upload_youtube.py` 的位置參數是 **slug（非路徑）**，用來推導
  `output/{slug}.md`（取標題/描述/標籤）與 `output/{slug}.mp4`。
- 後果：slug 變成一整串路徑 → 找不到 `.md`（metadata 全部 fallback 成 slug 字串）、
  且 mp4 路徑被二次拼接（`output/<整串路徑>.mp4`）→ 上傳找不到檔。
- 修正：改成 `upload_youtube.py {slug} --video <mp4> --output <dir> [--config ...] --privacy <p> [--dry-run]`。
  位置給乾淨 slug（讓 `.md` metadata 正確解析），再用 `--video` 明確指定成片路徑。

> 修正後四端 slug→檔名推導完全一致：`generate_script`(寫出) /
> `tts_pipeline`(`.voice.txt`→`.mp3`) / `make_video`(`--slug`自推) /
> `upload_youtube`(slug 推 `.md`/`.mp4`) 全部對齊 `output/{slug}.md/.voice.txt/.mp3/.mp4`。

### 【中等・YouTube metadata 正確性】

**問題 3：`upload_youtube.py` 解析 Hashtags 多吃到一個 `**` 假標籤**
- `.md` 那行是 `**Hashtags：** #自動交易 ...`，原正規式把冒號後的 `**`（markdown 粗體尾）
  也抓進來，產生一個內容為 `**` 的垃圾標籤送去 YouTube。
- 修正：每個 tag 多 `strip("#*")`，並過濾掉「不含任何文字字元」的純標點殘渣。
  修正後標籤為乾淨的 `自動交易, 派網, Pionex, 量化交易, 網格交易, 被動收入`。

### 【中等・Windows 中文相容】

**問題 4：`upload_youtube.py` 缺 UTF-8 主控台保護**
- Windows 主控台預設 cp950，`print_metadata()` 印中文標題/描述時會
  `UnicodeEncodeError` → **上傳前直接崩潰**。`run_all.py` 有做 stdout 重設、
  但子程式被 subprocess 叫起時 stdout 仍是 cp950。
- 修正：在 import 後加入 stdout/stderr `reconfigure(encoding="utf-8", errors="replace")`。

**問題 5：`make_video.py` 缺 UTF-8 主控台保護**
- 同上，dry-run 印中文 slug/段落小標時 cp950 會炸。
- 修正：同樣加入 UTF-8 stdout/stderr 重設。

### 【文件】

**問題 6：`run_all.py` 檔頭把子程式介面標成「假設」且寫錯**
- 原檔頭docstring 寫「假設位置參數吃 mp3/mp4」，與實際不符、易誤導後續維護。
- 修正：改寫成「實際 CLI 介面（已對齊）」，並標明 make_video 用 `--slug`、
  upload 位置是 slug + `--video`。

> 註：`generate_script.py`、`tts_pipeline.py` 為**既有產線**，介面確認無誤，
> 為維持既有產線穩定**未改動**（它們寫檔皆已正確 UTF-8，console 中文輸出由 `run_all.py`
> 包覆，且不影響成品檔案）。

### 安全性複查（無問題，維持原樣）

- `upload_youtube.py`：`--privacy` 預設 **private**，`publishAt` 一律強制
  `privacyStatus=private` 後再帶排程時間 ✓（符合 YouTube 規則）。
- `run_all.py`：`--stop-after` 預設 **video**（產出 mp4 就停，**不自動上傳**），
  要上傳必須明確 `--upload`；`--privacy` 預設 private ✓。
- 全程**未硬寫任何金鑰**；OAuth token 寫入 `token.json` 時**不印出內容**，
  `client_secrets.json` 內容也絕不外印 ✓。
- `videos.insert(part="snippet,status")` + `MediaFileUpload(resumable=True)` +
  `next_chunk()` 進度迴圈 + 5xx 指數退避重試 ✓ API 用法正確。
- `make_video.py` 無 `PEXELS_API_KEY` 時自動降級為 PIL 字卡投影片，**確實能產出 mp4** ✓；
  ffmpeg 依賴已在檔頭與 requirements 註明。
- `requirements.txt` 涵蓋全部新依賴（moviepy 透傳 Pillow/numpy；google 三件套；
  requests）且無多餘 ✓。

## ③ Carson 必須親自做的「憑證／金鑰申請」逐項清單

> 程式不會、也不該幫你申請金鑰。以下每項都要你本人操作一次。
> `--dry-run` 完全不需要任何金鑰，可先零成本驗證流程。

### (1) Anthropic Claude API Key — 用於 ① 產腳本
- **去哪申請**：<https://console.anthropic.com/> → 登入 → 「API Keys」→ Create Key。
- **長相**：`sk-ant-...`
- **放哪 / 怎麼設**（PowerShell，永久寫入用 `setx`，設完要開新視窗才生效）：
  ```powershell
  setx ANTHROPIC_API_KEY "sk-ant-你的金鑰"
  ```
- **不設會怎樣**：`generate_script.py` 自動 fallback 用內建模板出「骨架腳本」（要手填），不會中斷。

### (2) ElevenLabs API Key — 用於 ② 產配音
- **去哪申請**：<https://elevenlabs.io/> → 登入 → 右上頭像 → Profile → 複製 API Key。
- **放哪 / 怎麼設**：
  ```powershell
  setx ELEVENLABS_API_KEY "你的 elevenlabs 金鑰"
  ```
- **另外**：到 `channel_config.json` 的 `tts.voice_id` 把 `REPLACE_ME_CARSON_VOICE_ID`
  換成你選定的繁中 voice profile id（在 ElevenLabs 的 Voices 頁可取得）。

### (3) Pexels API Key — 用於 ③ 剪輯成片抓 B-roll 空鏡（選用）
- **去哪申請**：<https://www.pexels.com/api/> → 免費註冊 → Your API Key（免費、無限額）。
- **放哪 / 怎麼設**：
  ```powershell
  setx PEXELS_API_KEY "你的 pexels 金鑰"
  ```
- **不設會怎樣**：`make_video.py` 自動降級為深色漸層字卡投影片，**照樣產得出 mp4**，不會中斷。

### (4) ★ YouTube OAuth 憑證 — 用於 ④ 上傳（最費工，**只能你本人做**）

YouTube Data API 用的是 **OAuth 2.0（不是單純 API key）**，授權的是「你的 Google 帳號可以上傳到你的頻道」——
**這一步無法由程式或他人代勞，必須你本人在自己的瀏覽器登入並點同意**。流程如下：

1. **建立 Google Cloud 專案**
   <https://console.cloud.google.com/> → 建立新專案（例：`carson-quant-youtube`）。
2. **啟用 YouTube Data API v3**
   左側「API 和服務 → 程式庫」→ 搜尋「YouTube Data API v3」→ 啟用。
3. **設定 OAuth 同意畫面**
   「API 和服務 → OAuth 同意畫面」→ User Type 選「**外部**」→ 填 App 名稱／支援信箱 →
   在「**測試使用者 (Test users)**」加入**你自己上傳用的那個 Google 帳號**
   （未發布的應用只有測試使用者能授權，自用足夠，不必送審）。
4. **建立 OAuth 用戶端 ID**
   「API 和服務 → 憑證 → 建立憑證 → OAuth 用戶端 ID」→ 應用程式類型選「**桌面應用程式 (Desktop app)**」
   → 建立後按「**下載 JSON**」。
5. **放置憑證檔（放哪很重要）**
   把下載的 JSON **改名為 `client_secrets.json`**，放到專案根目錄：
   ```
   D:\carson-agent\youtube_channel\client_secrets.json
   ```
   （與 `channel_config.json` 同一層。程式預設就讀這個路徑。）
6. **★ 一次性瀏覽器授權（只能你本人做）**
   第一次跑到「真正上傳」（非 `--dry-run`）時，`upload_youtube.py` 會**自動開啟你的瀏覽器**，
   要你**登入步驟 3 加進測試使用者的那個 Google 帳號並點「允許」**。
   授權成功後 token 會快取到 `D:\carson-agent\youtube_channel\token.json`，
   **之後上傳就免再授權**（token 過期會自動 refresh）。

> **安全提醒（務必）**：`client_secrets.json`、`token.json`、`.env` 與所有金鑰
> **絕不要**上傳到 GitHub。若用 git，請全部加進 `.gitignore`。
> **配額**：YouTube Data API 每日預設 10,000 點，單支影片上傳約耗 1,600 點，自用足夠。

## ④ 「從題目到影片上架」的最終一行指令操作示範

> 一律先 `--dry-run` 驗證接線、再實跑；上傳一律先 `private` 內審、確認沒問題再轉 public。

### 步驟 0：環境就緒（每次開新視窗做一次）
```powershell
Set-Location D:\carson-agent\youtube_channel
.\.venv\Scripts\Activate.ps1
```

### 步驟 1：先全程 dry-run（零金鑰、不花錢、不上傳，驗證流程能不能跑通）
```powershell
python scripts\run_all.py "派網網格機器人新手設定＋回測驗證" --dry-run --stop-after upload
```
看到四階段全綠、結尾印出「✅ 流水線完成」即代表接線正確。

### 步驟 2：實跑到「成片」就停（預設行為，產出 .md/.voice.txt/.mp3/.mp4，**不上傳**）
```powershell
python scripts\run_all.py "派網網格機器人新手設定＋回測驗證"
```
> 預設 `--stop-after video`：產出 `output\{slug}.mp4` 後停下，**讓你先人工檢查成片**。

### 步驟 3：人工檢查 `output\{slug}.mp4` OK 後，先以 private 試上傳（安全內審）
```powershell
python scripts\run_all.py "派網網格機器人新手設定＋回測驗證" --upload
```
> `--upload` 才會真的上傳；`--privacy` 預設 **private**。首次會開瀏覽器要你 OAuth 授權（見 ③-(4)）。
> 上傳成功會印出 `videoId` 與 Studio 編輯網址。

### 步驟 4：到 YouTube Studio 確認影片內容無誤後，再轉公開
兩種方式擇一：
- **手動**：到 Studio（上一步印出的 `studio.youtube.com/video/<id>/edit`）把可見度改成「公開」。
- **排程自動公開**（隱私自動鎖 private，到時自動轉公開）：
  ```powershell
  python scripts\upload_youtube.py "{slug}" --publish-at 2026-06-20T09:00:00+08:00
  ```

> 安全鐵則：**永遠不要**第一次就直接 `--privacy public`。先 private/unlisted 內審，
> 確認標題、描述、派網聯盟連結（記得先把 `channel_config.json` 的 `REPLACE_ME_*` 換成真連結）
> 都正確後，再公開。
