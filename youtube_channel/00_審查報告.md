## ② 發現並修正了哪些問題

### 🔴 致命級（會讓整條產線跑錯／對不上）

1. **`channel_config.json` 整份是錯的頻道。**
   原本內容是「深夜冷知識研究所（冷知識／科普／都市傳說）」——與本專案「量化阿森｜Carson Quant（量化交易）」完全無關，是別的範本殘留。所有 `.md` 文件與 `tts_pipeline.py` 都是量化交易頻道，唯獨 config 對不上。
   **修正**：整份重寫為「量化阿森｜Carson Quant」，頻道名、受眾、語氣、CTA、聯盟連結（Pionex／Binance 佔位）、SEO 標籤、voice 設定全部對齊定位文件。

2. **`generate_script.py` 產出檔名 與 `tts_pipeline.py` 預期吃的檔名不一致。**
   - 原本：`generate_script.py` 產 `output/{slug}_script.md` + `output/{slug}_voiceover.txt`
   - 但 `tts_pipeline.py` 的檔名約定是吃 `xxx.voice.txt` → 出 `output/xxx.mp3`
   - 兩端對不上（`_voiceover.txt` 會被推導出 `xxx_voiceover.mp3` 這種怪檔名）。
   **修正**：`generate_script.py` 改為產出 `output/{slug}.md` + `output/{slug}.voice.txt`，**正好就是 `tts_pipeline.py` 預期的 `.voice.txt`**，兩端完全對齊。已實測 `.voice.txt` → `.mp3` 推導正確。

3. **`channel_config.json` 缺 `tts` 區塊，TTS 永遠用到公版 Rachel 英文 voice。**
   `tts_pipeline.py` 從 config 的 `tts` 鍵讀 `voice_id` 等設定，但原 config 沒有此區塊，導致即使指定 config 也讀不到頻道固定 voice，會 fallback 到 ElevenLabs 公版 Rachel（英文女聲，不符繁中顧問定位）。
   **修正**：新 config 加入 `tts` 區塊（`voice_id` / `model_id` / `max_chars` / `stability` / `similarity_boost` / `price_per_1k`）。

### 🟠 嚴重級（README 照著做會跑不起來）

4. **README 的 CLI 指令是虛構的，與實際程式不符。**
   README 寫 `python generate_script.py --slug ... --pillar ... --title ... --hook ...`、`python tts_pipeline.py --slug ...`——但實際程式：`generate_script.py` 是「題目當位置參數」+ `--config/--output/--api-key/--no-llm`；`tts_pipeline.py` 是「.voice.txt 路徑當位置參數」。完全對不上，Carson 照抄必報錯。
   **修正**：README 步驟 3／4 與附錄快速指令全部改成真實 CLI，並加上 `--dry-run` 先估成本、`scripts\` 路徑前綴。

5. **`requirements.txt` 不存在，但 README 叫人 `pip install -r requirements.txt`。**
   **修正**：建立 `requirements.txt`。並修正 README 內列的套件——原列 `elevenlabs>=1.0.0`，但 `tts_pipeline.py` 其實是用 `requests` 直打 REST API，**根本不需要 elevenlabs SDK**；補上選用的 `pydub`。

6. **`tts_pipeline.py` 路徑寫死絕對路徑 `D:/carson-agent/youtube_channel/output`。**
   定位文件 / CLAUDE.md 顯示專案原始位置是 `C:\Users\User\Downloads\carson-agent`，硬寫 D 槽路徑一旦搬移就壞，也與 `generate_script.py` 用 `pathlib` 從 `__file__` 推導的作法不一致。
   **修正**：改為 `PROJECT_ROOT = Path(__file__).resolve().parent.parent` 推導，與 `generate_script.py` 一致，搬移不壞。

7. **README 資料夾結構圖與實際不符。**
   原圖把 `.py` 畫在根目錄、輸出畫在 `output/audio//video//upload/` 子層；實際程式在 `scripts/`、輸出是扁平的 `output/{slug}.*`。
   **修正**：結構圖重畫，反映真實佈局（程式在 `scripts/`、產出扁平在 `output/`、補上 config 與各 `.md`）。

### 🟡 一致性／品質級

   原本 `--config` 預設 `None`，等於每次都得手動帶 `--config` 才會套用頻道 voice，否則 fallback 公版 voice。
   **修正**：`run()` 改成不指定時自動讀專案根目錄的 `channel_config.json`，help 文字同步更新。

9. **`generate_script.py` 的 `--help` 範例是冷知識頻道題目。**
   殘留 `"為什麼貓會發出呼嚕聲？"`、`"黑洞裡面到底有什麼"`、`"都市傳說：午夜的電梯"`、`"鐵達尼號"`——off-brand。
   **修正**：改為量化交易範例（90 天實測、派網網格教學、Triple SuperTrend、馬丁格爾回測）。

10. **README 的 pipeline 表與「腳本/配音稿分離」說明引用舊檔名。**
    **修正**：全部對齊為 `output/{slug}.md` / `output/{slug}.voice.txt` / `output/{slug}.mp3`，並明確標註「兩端檔名約定對齊」。

11. **`generate_script.py` 模組 docstring 仍寫舊檔名 `<slug>_script.md` / `<slug>_voiceover.txt`。**
    **修正**：docstring 更新為新檔名，並補一句說明「`.voice.txt` 正是 tts 預期的約定」。

12. **`tts_pipeline.py` docstring 範例路徑用了不存在的 `scripts_out/` 資料夾。**

### ✅ 經查「沒問題、不需改」的項目（已驗證）

- **Python 語法 / import / pathlib**：兩支程式 `py_compile` 通過、`--help` 正常、實際執行通過。系統 Python 是 **3.9**，程式雖用 `dict[str, Any]` 等 PEP585 新式泛型註解，但因有 `from __future__ import annotations`（延遲評估）而在 3.9 安全執行——已實測無誤。

## ③ 已知限制 / Carson 需要手動做的事

這些是**程式碼無法代勞、必須由 Carson 親自完成**的外部步驟：

1. **申請並填入 API 金鑰（必做，否則產線跑不動）**
   - `ANTHROPIC_API_KEY`：到 console.anthropic.com 申請（產腳本用）。沒有時程式會 fallback 出「骨架模板」，需手動填內容。
   - `ELEVENLABS_API_KEY`：到 elevenlabs.io 申請（產配音用）。沒有時只能 `--dry-run` 估算，無法真的產 mp3。
   - 設定方式見 README §3.3（建議用 `setx` 永久寫入或 `.env`）。

2. **建立並填入頻道固定的 ElevenLabs voice_id（強烈建議）**
   `channel_config.json` 的 `tts.voice_id` 目前是佔位 `REPLACE_ME_CARSON_VOICE_ID`。需到 ElevenLabs 選一個（或複製/訓練一個）繁中冷靜顧問男聲，把它的 voice_id 填進去，否則會用公版英文 Rachel。

3. **申請聯盟行銷帳號並替換佔位連結（影響變現，越早越好）**
   - Pionex 派網推薦碼／返佣連結 → 替換 `channel_config.json` 的 `REPLACE_ME_PIONEX_REF`，以及 `03_第一支影片腳本.md` 描述欄的 `[PIONEX_AFFILIATE_LINK]` / `[PIONEX_REF_CODE]`。
   - 同理 Binance / Bybit / OKX（`[BINANCE_AFFILIATE_LINK]` 等）。
   - 注意：config 用 `REPLACE_ME_*` 約定、03 腳本用 `[佔位符]` 約定——兩者服務不同輸出（config 驅動自動產的 CTA；03 是手寫成品的描述欄），各自替換即可，不需統一。

4. **（選用）安裝 ffmpeg**
   只有當單支配音稿超過 2500 字元被切多段、且你想要更精準的 seek 時才需要。否則不裝也能正常出 mp3（程式會自動回退 binary 串接）。

5. **剪輯 / 縮圖 / 上傳仍是人工**
   `④ 剪輯（CapCut/DaVinci）`、`⑤ 縮圖（Canva）`、`上傳（YouTube Studio）` 目前不在自動化範圍，需手動。`assets/` 下的 `brand//templates//bgm//stock/` 子資料夾需自行建立並放素材。

## ④ 給 Carson 的「明天就能開始」三步驟行動清單

> 前提：已安裝 Python 3.9+。以下用第一支影片（選題清單 #1）走一遍。

**第 1 步：環境就緒（約 15 分鐘，只做一次）**
```powershell
Set-Location D:\carson-agent\youtube_channel
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
# 申請並設定金鑰（重開新視窗才生效）
setx ANTHROPIC_API_KEY  "sk-ant-你的金鑰"
setx ELEVENLABS_API_KEY "你的 elevenlabs 金鑰"
```
同時：去 ElevenLabs 選好繁中顧問男聲，把 voice_id 填進 `channel_config.json` 的 `tts.voice_id`；去申請 Pionex 推薦碼，替換 config 裡的 `REPLACE_ME_PIONEX_REF`。

**第 2 步：產第一支的腳本 + 配音（約 10 分鐘）**
```powershell
# 直接用已寫好的成品腳本 03，或用程式產一份新的：
python scripts\generate_script.py "我把 10 萬丟給自動交易機器人跑 90 天，結果出乎意料"
# → 打開 output\*.md 人工校稿（數據、Hook、CTA、聯盟連結）
# → 先 dry-run 估成本，再正式產配音
python scripts\tts_pipeline.py "output\我把 10 萬丟給自動交易機器人跑 90 天，結果出乎意料.voice.txt" --dry-run
python scripts\tts_pipeline.py "output\我把 10 萬丟給自動交易機器人跑 90 天，結果出乎意料.voice.txt"
```
（也可直接拿 `03_第一支影片腳本.md` 的逐字稿貼成 `.voice.txt` 餵 TTS——它已是成品級。）

**第 3 步：錄畫面 → 剪輯 → 上傳（半天內可完成第一支）**
- 螢幕錄製 Pionex／TradingView 操作畫面 + 用 Chart.js/DaVinci 做損益曲線動圖（依 03 腳本的【畫面】指示）。
- CapCut/DaVinci 把 `output\*.mp3` 拉進時間軸，對齊畫面與字卡。
- Canva 做縮圖（`10萬 → 90天` 大字），把 `03` 的描述欄/標籤貼進 YouTube Studio，放上 Pionex 聯盟連結，發佈。
