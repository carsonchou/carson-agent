# 量化阿森 YouTube 工作室｜公司章程（9 部門全自動運營）

> 老闆：Carson　|　願景：把「量化阿森｜Carson Quant」做到華語量化交易頻道**產業龍頭**
> 第一階段目標：**達成 YPP 營利門檻**（1000 訂閱＋4000 小時觀看／或 Shorts 90 天 1000 萬次觀看）
> 鐵則：① 內容**100% 誠實**，絕不編造個人損益或保證收益（會被砍頻道）② 每日高產出 ③ 全自動，總監管每日匯報老闆

---

## 🎯 達標策略（為什麼這樣排部門）

YPP 兩條路，**Shorts 路最快**：
- 長片路：1000 訂閱 + 4000 小時觀看 → 慢（要長停留）
- **Shorts 路：90 天 1000 萬次觀看 → 快（演算法推力大、製作成本低）** ← 主攻
- 策略：**Shorts 衝量拉曝光與訂閱 → 長片承接深度與觀看時數 → 雙線達標**

## 🏢 16 大部門（職責 × 每日產出 × 交接）

### ① 影片部門（長片）
- **職責**：產出 8–12 分鐘長片（策略拆解／工具教學／市場觀念／回測）。
- **每日產出**：1 支長片完整腳本（`output/{slug}.md` + `.voice.txt`）→ 配音 → 成片 → 縮圖。
- **交接**：題目來自③，SEO 來自⑤，成片交⑨上架。

### ② Shorts 部門（短影音，主攻達標）
- **職責**：產出 15–60 秒直式短影音（1080x1920），一個觀念一支、鉤子前 2 秒。
- **每日產出**：**3–5 支** Shorts 腳本 → 配音 → 直式成片 → 封面。**衝量是核心 KPI。**
- **交接**：題目來自③（含長片切片），交⑨上架。

### ③ 創作靈感部門
- **職責**：選題引擎。蒐集熱點、關鍵字、競品缺口，產出題庫與內容日曆。
- **每日產出**：當日長片題 1 + Shorts 題 3–5 + 維護 30 天內容日曆。
- **交接**：餵題給①②。

### ④ 頻道整理部門
- **職責**：頻道門面與資訊架構——播放清單、章節、片尾資訊卡、置頂留言、頻道介紹。
- **每日產出**：把新片歸入正確播放清單、維護一致品牌、檢查 metadata 一致性。

### ⑤ 流量部門（SEO／演算法）
- **職責**：標題、標籤、描述、縮圖文案的點擊與搜尋優化；關鍵字研究；A/B 縮圖建議。
- **每日產出**：當日所有新片的 SEO 包；回頭優化舊片標題/標籤。

### ⑥ 宣傳部門
- **職責**：站內社群貼文、跨平台導流文案（合規，不買量不spam）、社群互動企劃。
- **每日產出**：社群 Tab 貼文 1–2 則、可跨發的短文案。

### ⑦ 數據分析部門
- **職責**：每日拉頻道與影片數據（觀看、觀看時數、CTR、訂閱、Shorts 觀看），算出**距離 YPP 還差多少**，找出表現最好/最差的內容給③⑤調整。
- **每日產出**：數據儀表板 + 達標進度 + 行動建議。

### ⑧ 社群留言部門
- **職責**：抓新留言、草擬回覆（理性顧問口吻、不亂承諾）、找出可變內容的問題餵③。
- **每日產出**：留言回覆草稿 + 觀眾問題清單。

### ⑨ 審核部門（發布閘門｜代替老闆把關）
- **職責**：每支影片**發布前自動品管 + 誠信把關**，PASS 才准上架，FAIL 隔離不發、記錄原因。老闆不必逐支審。
- **檢查**：①技術（檔案完整、有影音軌、片長合理、Shorts<60s）②誠信禁語（保證賺/穩賺不賠…，且能辨識「不保證」這種誠實否定句不誤判）③合規（有標題、有風險聲明）。
- **每日產出**：審核通過清單 + 隔離清單（交⑩上架/退回重修）。實作：`scripts/audit_video.py`。

### ⑩ 總監管部門
- **職責**：統籌調度、每日彙整各部門產出、追蹤達標進度。
- **每日產出**：**《每日營運匯報》交給老闆 Carson** → 存 `STUDIO/REPORTS/`。

### ⑪ 決策部門（閉環大腦｜讓工廠越跑越聰明）
- **職責**：每日拉成效數據 → 分析 → **做營運決策** → 改寫各部門下一輪指令。加碼會紅的、砍掉沒人看的、發現方向就轉。
- **分工**：數據部門報「發生什麼」，決策部門決定「所以怎麼做」，總監管執行+匯報。營運層決策(轉題/改格式)自己拍板；花錢、大轉向才上呈老闆。
- **每日產出**：`STUDIO/production_orders.json`（餵補產部門偏向高效題材）+ `STUDIO/REPORTS/{date}_決策.md`。實作：`scripts/decision_dept.py`。
- **註**：細數據(CTR/觀看時數/留存)需接 YouTube Analytics 權限才更準；目前用基本統計，數據累積後第 2-4 週發揮全力。

### ⑫ 回顧檢討部門（自省閉環｜每輪跑完自動檢討並優化）
- **職責**：每輪產線跑完後**自我檢討**——收集真實訊號（產量／審核過審率＋失敗原因／上架數／頻道數據與昨日 delta／異常日誌），分析後**產出具體優化並自動套用**回各部門下一輪指令，讓工廠越跑越好。
- **每日產出**：`STUDIO/REPORTS/{date}_回顧檢討.md`（檢討報告）＋ 寫回 `boss_directives`（【自省優化】標籤）與 `production_orders`（調整多做／避免題材）＋ `metrics_history.json`（每日快照算成長）。實作：`scripts/retro_dept.py`，排程 09:37（上架後）。
- **誠實鐵則**：數據薄時給務實方向（衝量、累積數據），不掰假洞察。

### ⑬ 人事部（部門監察官 ＋ 編制管理）
- **職責**：盯的是「**其他部門本身的健康**」而非影片。① 出勤：各部門有沒有按排程跑；② 健康：誰一直出錯／連續失敗／產壞檔；③ 考核：產量、過審率、員額；④ 編制管理：老闆可在決策中心人事部分頁 **➕招募／➖ 分配各部門員額**來擴大公司。
- **員額即產能（誠實、非虛榮）**：員額存於 `STUDIO/headcount.json`；①影片／②Shorts 的員額＝每日產出量（`produce_batch` 讀取，加員額＝加產能），其餘部門為容量編制。
- **每日產出**：`STUDIO/REPORTS/{date}_人事監察.md` ＋ `STUDIO/hr_status.json`。實作：`scripts/hr_dept.py`。

### ⑭ 財務／變現部
- **職責**：損益與 ROI。收入（Pionex 返佣／YouTube 廣告，**無 API 需手動『記一筆』**）− 成本（目前全免費棧≈NT$0，唯 Anthropic API 為潛在成本）＝淨利。
- **每日產出**：`STUDIO/REPORTS/{date}_財務.md`＋`STUDIO/finance.json`。實作：`scripts/finance_dept.py`。決策中心「💰 記一筆帳」輸入收支。

### ⑮ 縮圖／CTR 部
- **職責**：縮圖覆蓋率檢查 + A/B 標題/縮圖文案優化建議。**誠實**：真 CTR 需 YouTube Analytics 權限（目前未授權），故以覆蓋率＋建議替代。
- **每日產出**：`STUDIO/REPORTS/{date}_縮圖CTR.md`。實作：`scripts/thumbnail_dept.py`。

### ⑯ 競品情報部
- **職責**：用 YouTube Search 掃同類頻道近期高觀看影片，抽標題公式與熱度餵 ③靈感／⑪決策。
- **每日產出**：`STUDIO/REPORTS/{date}_競品情報.md`＋`STUDIO/intel.json`。實作：`scripts/intel_dept.py`。

> 註：④頻道整理(`organize_dept.py` 自動歸播放清單)、⑥宣傳(`promo_dept.py` 跨平台文案草稿)、⑧社群留言(`comment_dept.py` 回覆草稿)已從「規劃中」做成真自動化；⑥⑧只產草稿不自動發（誠信＋防 spam）。
> 編制：員額存 `STUDIO/headcount.json`，人事部可 ➕招募／🚀自動擴編；①②員額＝每日產量。全公司可一鍵壓榨／一鍵最大化壓榨（寫加碼令給決策部門）。

## 🔄 每日自動化流程（全自動閉環，審核部門代替老闆把關）

```
⑪決策(拉昨日數據→寫生產指令) → ③靈感(照指令選題) →
①影片+②Shorts(腳本→免費配音→成片→縮圖) → ⑤流量(SEO) →
⑨審核部門(品管+誠信，PASS才放行) → 自動公開上架(每日≤6支,Shorts優先) →
④整理(歸類) → ⑥宣傳(貼文) → ⑦數據(拉成效) → ⑧留言(回覆) →
⑩總監管(彙整→每日匯報給老闆) → ⑫回顧檢討(自省→優化下一輪)
```
排程：05:37 決策 → 06:07 補產 → 09:07 審核上架 → 09:37 回顧檢討（Windows 工作排程器，永久自動）。
> 老闆只看每日匯報，不必逐支審核也不必逐日決策——審核部門把關、決策部門掌舵。

## ⚙️ 產能上限與成本（誠實）

- **配音（ElevenLabs）**：免費版每月 1 萬字元 ≈ 撐不起每日量。**每日高產出需付費方案**（這是唯一會「影響產出」的成本，依老闆鐵則應升級）。
- **腳本生成**：由排程的 AI 代理（Claude）每日自動撰寫，無需額外金鑰；或設 ANTHROPIC_API_KEY 走 run_all.py。
- **帳號**：已通過電話驗證（可上自訂縮圖）✓。
